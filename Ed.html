Edward the Personal Desk Assistant

Overview

Edward is a spherical personal assistant robot using APIs from OpenAI, Deepgram, and Google TTS to transcribe speech, generate a logical answer, and return a humanlike response. Edward utilizes an ESP32-WROOM board with SD card, micropone, and speaker modules on a custom made PCB. Three servos direct the semi-sphere head to tilt forward, sideways, or spin along the tilt axis to convey emotions.

Build Details

  • Spherical body for a friendly looking robot
  • Two blue LEDs serve as eyes
  • A joystick inspired mechanism that directs all head movement.
  • All of the APIs can be run at minimal cost (I have spent less than 20 cents in API costs throughout testing and using Edward).
  • All of the actual components and materials cost roughly $75

Future Implementation

    The original version of Edward had a few drawbacks which I plan on addressing with the newer "Archibald."

  • For one, the latency caused by the daisy chain of API keys was relatively high, ranging from 5-10 seconds depending on the length of the prompt and the strength of the Wifi signal
  • Secondly, I had not accounted for the noise, both electrical and mechanical, of the moving servo motors, and how they would interfere with the microphone input. Because of it, Edward will occasionally misunderstand prompts or not hear them altogether.

How I am addressing these shortcomings

  • To address the latency issue, I am upgrading the circuitry to be based on a Raspberry Pi 4b, utilizing the newer Realtime API by OpenAI. This API, which gained prominence nearing the end of my building of Edward, offers speech to speech communication between a person and the OpenAI agent. The latency as I have tested so far, tends to be under a second, even with lower wifi connectivity.
  • For the noise issues I have decided to drop the servos altogether. I am going to be replacing them with an LED cube, which will be able to express colors and emotions through the Realtime API's function calling ability. While the LEDs will still cause a significant amount of electrical noise, I plan on using some sort of shield board between the Raspberry pi and the cube to reduce its interference, or add a noise reducing circuit using a low-pass filtering circuit.
